# IPL Win Intelligence Platform Documentation

## Project Overview
This platform is a comprehensive **IPL Win Probability and Match Simulation** tool. It allows users to simulate IPL matches in real-time, analyze win probabilities using machine learning, and explore the impact of specific events (like a wicket or a six) on the final match outcome.

## How it Helps with Data Analysis

### 1. Real-time Win Probability (XGBoost)
The core of the platform is an **XGBoost Classification model** trained on over 290,000 historical IPL deliveries. 
- **Situational Analysis**: The model factors in the current run rate, required run rate, wickets fallen, balls remaining, and the specific match phase (Powerplay, Middle, Death).
- **Dynamic Adjustments**: It dynamically adjusts probabilities based on "extra-tabular" features like:
    - **Toss Impact**: Historical advantage of the toss winner (e.g., chasing advantage in certain seasons).
    - **Player Impact**: Dynamic aggression/economy indices for specific batters and bowlers.
    - **Phase Impact**: Volatility analysis for different stages of the innings.

### 2. Match Simulation & Re-simulation
The platform features a "Zero-Gravity Timeline" that allows for deep counterfactual analysis:
- **Ball Editing**: You can change any past delivery's outcome (e.g., change a '6' to a 'W') and instantly see how the entire match state and win probability history recalculate.
- **Strike Rotation Logic**: Accurate logic for batter strike rotation on odd runs and over ends.
- **Innings Transition**: Seamless transition between 1st and 2nd innings with target calculation.

### 3. Visual Data Analytics
- **Dual-Timeline Graph**: A live `recharts` graph showing the win probability evolution for both teams ball-by-ball.
- **Phase Impact Visualization**: Interactive bars showing the "volatility" and importance of the current match phase.
- **AI Match Insights**: Natural language explanations generated by the backend to explain *why* the probability shifted (e.g., "A boundary here shifts win chance by +5.2%").

## Technical Stack
- **Backend**: FastAPI (Python), XGBoost, Pandas, Joblib.
- **Frontend**: React (Vite), Tailwind CSS, Framer Motion, Recharts, Lucide-Icons.
- **Data Source**: Historical IPL dataset (2008-2024).

## Deployment Guide

### Option 1: Dual Deployment (Recommended)
Since InfinityFree only supports PHP, you must split the hosting:

#### 1. Backend (Python/FastAPI)
Host the backend on a service that supports Python (e.g., **Render**, **Railway**, or **Koyeb**).
- Upload the `backend/` folder to GitHub.
- Connect the repository to Render.
- Render will automatically use the `Procfile` and `requirements.txt`.
- Set the `PORT` environment variable if needed.
- Note your backend URL (e.g., `https://ipl-api.onrender.com`).

#### 2. Frontend (React/Vite)
Host the static frontend on **InfinityFree**.
- Create a `.env.production` file in `frontend/`:
  ```
  VITE_API_URL=https://your-backend-url.onrender.com/api
  ```
- Run the build command:
  ```bash
  cd frontend
  npm run build
  ```
- Upload the contents of the `frontend/dist/` folder to the `htdocs/` directory of your InfinityFree account via FTP.
- The included `.htaccess` file will handle React Router paths automatically.

### Option 2: All-in-one (Vercel/Netlify)
You can also host both on Vercel by using Vercel Functions for the Python backend, but splitting is often easier for long-running ML processes.
